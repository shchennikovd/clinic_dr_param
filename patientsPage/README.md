# 👥 Пациентам

> Коммерческая задача по разработке страницы **«Пациентам»** для сайта частной медицинской клиники.

---

# 📖 Описание

Страница представляет собой раздел с карточками-ссылками, обеспечивающими быстрый доступ к основным направлениям и сервисам клиники.

Задача выполнена в рамках коммерческой разработки по предоставленному дизайн-макету.

---

# 🎯 Цель задачи

Разработать адаптивную страницу с карточками-ссылками в соответствии с дизайн-макетом и подготовить ее для последующей интеграции в существующий коммерческий проект.

---

# 🛠 Что реализовано

* адаптивная верстка страницы;
* карточки-ссылки с полностью кликабельной областью;
* семантическая HTML-разметка;
* подготовка структуры страницы для подключения общих компонентов проекта (`header.php` и `footer.php`);
* организация стилей с разделением на глобальные настройки, переменные и стили страницы;
* использование CSS Grid для построения адаптивной сетки карточек;
* подготовка страницы к интеграции в существующий коммерческий проект.

---

# ⚙️ Особенности реализации

Во время разработки были учтены следующие особенности проекта:

* самостоятельно спроектировано поведение интерфейса между Desktop и Mobile, поскольку макеты предоставлены только для разрешений **1920px** и **360px**;
* реализован адаптивный переход сетки карточек с двух колонок на одну;
* использован `clamp()` для плавного масштабирования отдельных элементов без необходимости создавать дополнительные брейкпоинты;
* карточки реализованы как переиспользуемый компонент. По техническому заданию требовалось сверстать один типовой элемент, поскольку остальные отличаются только содержимым. Верстка изначально спроектирована таким образом, чтобы корректно отображать любое количество карточек без изменения HTML-структуры или CSS;
* структура CSS разделена на глобальные стили, переменные, шрифты и стили страницы для повышения читаемости и удобства сопровождения;
* при разработке учитывалась последующая интеграция страницы в существующий коммерческий проект.

# 📂 Структура директории

```text
patientsPage/
├── assets/
│   ├── img/
│   ├── fonts.css
│   ├── globals.css
│   ├── style.css
│   └── variables.css
├── Dockerfile
├── header.php
├── footer.php
├── index.php
└── script.js
```

### Назначение файлов

| Файл            | Назначение                                                                                                                                                                                          |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `index.php`     | Основная структура страницы и подключение компонентов.                                                                                                                                              |
| `header.php`    | Точка подключения общей шапки проекта. В демонстрационной версии используется заглушка соответствующей высоты.                                                                                      |
| `footer.php`    | Точка подключения общего подвала проекта.                                                                                                                                                           |
| `style.css`     | Стили страницы «Пациентам».                                                                                                                                                                         |
| `globals.css`   | Общие стили проекта.                                                                                                                                                                                |
| `variables.css` | CSS-переменные (цвета, размеры, типографика).                                                                                                                                                       |
| `fonts.css`     | Подключение используемых шрифтов.                                                                                                                                                                   |
| `script.js`     | JavaScript для интерактивных элементов страницы (при наличии).                                                                                                                                      |
| `Dockerfile`    | Конфигурация Docker-контейнера для публикации демонстрационной версии страницы на Render. Используется для предоставления заказчику доступа к результату разработки без локального запуска проекта. |

# ⚠️ Особенности демонстрационной версии

Полная кодовая база коммерческого проекта недоступна и не публикуется.

По этой причине в демонстрационной версии отсутствует оригинальная шапка сайта. Вместо нее используется временный блок, соответствующий размерам настоящего `header`, что позволяет сохранить корректную структуру страницы и расположение контента относительно оригинального макета.

После интеграции в основной проект этот блок заменяется настоящей шапкой сайта.
