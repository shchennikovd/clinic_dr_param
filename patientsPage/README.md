# 👥 Пациентам

> Коммерческая задача по разработке страницы **«Пациентам»** для сайта частной медицинской клиники.

---

# 📖 Описание

Страница представляет собой раздел с карточками-ссылками, обеспечивающими быстрый доступ к основным направлениям и сервисам клиники.

Задача выполнена в рамках коммерческой разработки по предоставленному дизайн-макету.

---

# ⚠️ Особенности демонстрационной версии

В репозитории опубликована демонстрационная версия страницы.

Полная кодовая база коммерческого проекта недоступна и не публикуется, поэтому в демо отсутствует оригинальная шапка сайта. Вместо нее используется временный блок соответствующей высоты, позволяющий сохранить корректное расположение контента относительно дизайн-макета.

После интеграции в основной проект данный блок заменяется настоящим `header` сайта.

---

# 🌐 Демонстрация

Страница развернута на платформе **Render** и доступна для просмотра без локального запуска проекта.

**Live Demo**

> https://clinic-dr-param-patientspage-gop4.onrender.com/

---

# 🎯 Цель задачи

Разработать адаптивную страницу с карточками-ссылками в соответствии с дизайн-макетом и подготовить ее для последующей интеграции в существующий коммерческий проект.

---

# 🛠 Что реализовано

- адаптивная верстка страницы;
- карточки-ссылки с полностью кликабельной областью;
- семантическая HTML-разметка;
- подготовка структуры страницы для подключения общих компонентов (`header.php` и `footer.php`);
- организация CSS с разделением на глобальные стили, переменные и стили страницы;
- использование CSS Grid для построения адаптивной сетки карточек;
- подготовка страницы к интеграции в существующий коммерческий проект.

---

# ⚙️ Особенности реализации

- Самостоятельно спроектировано адаптивное поведение между **Desktop (1920px)** и **Mobile (360px)**, поскольку промежуточные макеты отсутствовали.
- Реализован адаптивный переход сетки карточек с двух колонок на одну.
- Использован `clamp()` для плавного масштабирования отдельных элементов без создания дополнительных брейкпоинтов.
- Карточки реализованы как переиспользуемый компонент, позволяющий отображать любое количество элементов без изменения HTML-разметки или CSS.
- Структура CSS разделена на глобальные стили, переменные, шрифты и стили страницы для удобства сопровождения.
- При разработке учитывались требования существующей кодовой базы и возможность безопасной интеграции страницы в коммерческий проект.

# 📂 Структура директории

```text
patientsPage/
├── assets/
├── header.php
├── footer.php
├── index.php
├── script.js
├── Dockerfile
└── README.md
```

### Назначение основных файлов

| Файл | Назначение |
|------|------------|
| `index.php` | Основная структура страницы. |
| `header.php` | Точка подключения общей шапки проекта (в демо используется заглушка). |
| `footer.php` | Точка подключения общего подвала проекта. |
| `style.css` | Стили страницы. |
| `globals.css` | Общие стили проекта. |
| `variables.css` | CSS-переменные проекта. |
| `fonts.css` | Подключение используемых шрифтов. |
| `script.js` | JavaScript интерактивных элементов страницы (при наличии). |
| `Dockerfile` | Используется для развертывания демонстрационной версии страницы на Render. |
